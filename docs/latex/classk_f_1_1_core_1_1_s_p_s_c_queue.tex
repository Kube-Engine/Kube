The S\+P\+SC queue is a lock-\/free queue that only supports a Single Producer and a Single Consumer The queue is really fast compared to other more flexible implementations because the fact that only two thread can simultaneously read / write means that less synchronization is needed for each operation. The queue supports ranged push / pop to insert multiple elements without performance impact.

\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Type$>$\newline
class k\+F\+::\+Core\+::\+S\+P\+S\+C\+Queue$<$ Type $>$}

The S\+P\+SC queue is a lock-\/free queue that only supports a Single Producer and a Single Consumer The queue is really fast compared to other more flexible implementations because the fact that only two thread can simultaneously read / write means that less synchronization is needed for each operation. The queue supports ranged push / pop to insert multiple elements without performance impact.
